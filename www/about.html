About

This server is built entirely on Go's net package. It opens a raw TCP listener, accepts connections, and handles each one in its own goroutine.

How It Works

  1. A TCP listener binds to port 8080
  2. Incoming connections are accepted in a loop
  3. Each connection is handled concurrently in a goroutine
  4. Raw bytes are read until the end-of-headers marker (\r\n\r\n)
  5. The request line and headers are parsed manually
  6. The requested file is loaded from disk and served with the correct content type

Why?

This project demonstrates how HTTP works under the hood. By skipping Go's built-in net/http server, every part of the protocol is handled explicitly — from parsing request lines to formatting response headers.
